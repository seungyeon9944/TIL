### Pinia
Vue 공식 상태 관리 라이브러리로, 여러 컴포넌트가 함께 사용해야 하는 공통 데이터를 중앙 저장소에서의 통합 관리를 도와줌

Pinina 구성요소
### 1. store
- 공통 데이터 관리하는 중앙 저장소
- 모든 컴포넌트가 공유, 기능이 작성됨
- `defineStore()`의 반환값(store)을 담는 변수 이름은 use ... Store 패턴 사용하기 

### 2. state
- 중앙 저장소에 저장되는 반응형 상태 (데이터)
- 해당 값 (count)를 변경하면, 데이터 사용하고있는 모든 컴포넌트의 화면이 알아서 업데이트됨
- **ref( )**와 같은 역할

### 3. getters
- 계산된 값
- state를 기반으로 파생된 값 정의
- **computed( )**와 같은 역할

### 4. actions
- state 변경
- **methods**와 같은 역할

### 5. 반환 값
- store에서는 공유하지않는 **private한 상태 속성 가지지않음**

### 6. plugin
- 상태관리에 필요한 추가 기능 제공하거나 확장하는 도구나 모듈

### Local Storage
브라우저 내에 key-value 쌍을 저장하는 웹 스토리지 객체
- 페이지 새로고침하고 브라우저 다시 실행해도 데이터 유지
- **쿠키**와 다르게 네트워크 요청 시 서버로 전송 X
- 여러 탭이나 창 간에 데이터 공유 가능
- 목적 : 웹 애플리케이션에서 **사용자 설정, 상태 정보, 캐시 데이터 등을 클라이언트 측에서 보관**하여 **웹사이트 성능 향상, 사용자 경험 개선**
